AndroidStudio快速导入Eclipse工程--2/2

By junwenZhang

发表于 2015-08-09

前言

上一次Android Studio导入YunOS Eclipse工程遗留一个问题：如何在导入之后还可以保持原有的目录结构，方便我们使用Git进行代码版本管理，接下来我们就来解决这个问题
Android Studio版本：1.3

详细步骤

1. 打开一个已经存在的android项目（eclipse创建的）

2. 进行Project设定


设定SDK版本和OUT目录

3. 进行Module设定

由于直接打开的Module不能够自动识别dependence关系，需要重新导入一遍module
删除掉原有Module

导入Module，即原有Eclipse工程目录，按照提示直接进行下一步

取消HWDroid，导入的HWDroid Module会自动进行依赖，如果这个地方勾选的话会产生重复依赖

4. 运行设定

导入后产生两个Module，Gallery 依赖HWDroid module，这两个Module 的SDK版本要设定为一样
进行运行设定，Module选择Gallery

5. 签名设定

custom debug keystore 中导入签名文件

6. 忽略某些文件的编译

点击运行，可能会提示错误，HWDroid的R文件编译出错，暂时忽略掉它

如果一切顺利的话，接下来就可以编译运行调试程序了

后续

使用Android Studio之后会产生一些配置文件，在用Git进行代码版本管理之前，记得把这些文件加入.gitignore中去哦，接下来你就可以愉快的使用Android Studio 编码了

AndroidStudio快速导入Eclipse工程---1/2

By junwenZhang

发表于 2015-06-28

前言

目前Android studio已经成为google主推的IDE工具，同时也有很多方便的工具可以供使用，例如memory monitor，cpu info等等，本文主要讲述如何将Eclipse工程导入Android studio, 以Gallery相册为例，先行记录下来，避免忘记。因时间有限，遗留了很多问题，希望一起讨论解决

详细步骤

1.下载安装Android Studio

http://www.android-studio.org/
http://developer.android.com/sdk/index.html

2.导入Eclipse工程


按照提示进行下一步，Android Stuido会将当前的Eclipse project进行copy到新的路径下，原有的依赖关系会自动设置，无需担心

3. 配置修改Project

  • 编译时不识别java-symbol，暂时注释(不影响运行调试)
  • 因为多个 jar 包里包含了同样的文件（NOTICE.txt/LICENSE.txt），导致打包时因为担心相互覆盖问题而提示出错,所以需要在Module Gallery的build.gradle文件中加入红色部分

  • 可能是因为AS默认只导入lib目录下的jar包，子文件夹下jar不能自动导入，所以需要拷贝自动导入时缺失的两个lib文件TA4AIDL.jar、tyid.jar到新生成的Android stuido project目录中，以我的为例是：
    ~/workspace/Gallery/gallery/libs,并在Module Gallery的build gradle文件中加入

  • 增加签名。Module gallery上点击右键选择open module settings，按照下图进行设置

    选择设置好的签名

  • 运行设置。菜单栏Run中点击Edit configuration，选择Target Device为Usb device

遗留问题

1. Android Studio 是将工程文件copy到一个新的地方进行修改，Eclipse project和Android studio无法共同修改同一份代码

2. Android Studio 编译时不识别java-symbol的问题需要继续跟进

3. 集成Git进行代码版本管理

Mac下基于Github搭建个人Blog

By junwenZhang

发表于 2015-06-17

记录一下在Mac下机遇Github搭建个人blog的过程
主要参考如何搭建一个独立博客——简明Github Pages与Hexo教程
其中需要注意的地方有

  • _config.yml文件的配置应该为
    deploy:
    type: git
    repo: https://github.com/junwenZhang/junwenZhang.github.io.git
    branch: master
    上述repo的方式为https提交，每次hexo 部署时都需要输入用户名和密码，比较麻烦，推荐使用下面的配置
    下面的方式为ssh提交，无须输入用户名和密码
    deploy:
    type: git
    repo: git@github.com:junwenZhang/junwenZhang.github.io.git
    branch: master
  • Hexo 部署失败
    hexo deploy命令显示ERROR Deployer not found : git
    请运行下npm install hexo-deployer-git —save
    再hexo g，hexo d即可

JPG Flash Flag

By junwenZhang

发表于 2015-06-16

相册中的闪关灯的开关值是通过exif parse出来的,Exif 2.1标准中通过判断返回值为0则认为关闭了闪关灯
但是Exif 2.2 标准中规定了有3个值可以认为是关闭了闪关灯，见下图代码
ExifInterface exif = new ExifInterface(mFilePath);
String tmp = exif.getAttribute(ExifInterface.TAG_FLASH);
//Exif 2.2 Specifies the resulting Flash tag values
//0000.H = Flash did not fire
//0010.H = Flash did not fire, compulsory flash mode
//0018.H = Flash did not fire, auto mode
